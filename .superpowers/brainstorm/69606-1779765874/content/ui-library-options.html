选择 UI 组件库方案

配合 Tailwind CSS 使用，替换现有内联 style 方案

A

shadcn/ui + Tailwind

复制源码到项目，完全可定制，无 npm 版本锁定。组件质量高（Radix UI 底层），暗色主题开箱即用。目前最流行的方案。

优势

  • 源码完全在项目里，可随意改
  • Radix 底层保障无障碍/键盘导航
  • 暗色主题 + CSS 变量，与现有 token 体系兼容
  • 社区生态最活跃

劣势

  • 首次安装需手动 add 各组件
  • 学习曲线（需理解 cn() 工具函数）
B

Headless UI + Tailwind

Tailwind 官方出品的无样式组件库。只提供交互行为（dropdown/dialog/listbox），样式完全由你用 Tailwind 编写。

优势

  • 官方出品，与 Tailwind 最匹配
  • 行为层与样式层完全解耦
  • 包体积很小

劣势

  • 只有少量组件（~10 个）
  • 需要自己写所有样式，工作量较大
  • 复杂组件（Table/DatePicker）需自己做
C

Radix UI Primitives + Tailwind

直接使用 Radix Primitives（shadcn 底层），搭配 Tailwind 自定义样式。比 shadcn 更底层、更灵活，但更多手工配置。

优势

  • 无障碍和键盘支持最完善
  • 组件数量多（覆盖所有交互需求）
  • 无预置样式，零约束

劣势

  • 需要为每个组件手写全套 Tailwind 样式
  • 相比 shadcn 工作量更大
  • 社区模板资源不如 shadcn 丰富
推荐

推荐 方案 A（shadcn/ui）——既保留了 Radix 的无障碍底层，又提供了开箱即用的暗色主题和 CSS 变量系统，与现有的 --accent/--bg-card token 体系无缝对接，迁移成本最低。
